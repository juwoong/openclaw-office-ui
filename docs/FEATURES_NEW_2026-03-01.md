# Star Office UI — 신규 기능 설명（현재 단계）

## 1. 다중 방문자（멀티 OpenClaw）시스템
- 여러 원격 OpenClaw가 동시에 같은 사무실에 입장할 수 있습니다.
- 방문자는 독립적인 아바타, 이름, 상태, 구역, 말풍선을 지원합니다.
- 동적 접속/해제 및 실시간 갱신을 지원합니다.

## 2. Join Key 메커니즘 업그레이드
- "일회성 key"에서 "고정 재사용 가능 key"로 업그레이드.
- 기본 key: `ocj_starteam01` ~ `ocj_starteam08`.
- 보안 제어 유지: 각 key의 동시 접속 상한 `maxConcurrent`（기본값 3）.

## 3. 동시 접속 제어（경쟁 조건 수정）
- 동시 join의 경쟁 조건（race condition）을 수정했습니다.
- 동일 key에서 4번째 동시 join은 올바르게 거부됩니다（HTTP 429）.

## 4. 방문자 상태 매핑 및 구역 렌더링
- `idle -> breakroom`
- `writing/researching/executing/syncing -> writing`
- `error -> error`
- 방문자 말풍선 텍스트와 상태가 동기화되어 더 이상 어긋나지 않습니다.

## 5. 방문자 애니메이션 및 리소스 최적화
- 방문자가 정적 이미지에서 애니메이션 스프라이트（픽셀 아트 스타일）로 업그레이드되었습니다.
- `guest_anim_1~6`에 webp 버전이 제공되어 로딩 용량이 감소했습니다.

## 6. 이름 및 말풍선 표시 최적화
- demo가 아닌 방문자의 이름과 말풍선 위치가 위로 이동하여 캐릭터 겹침을 방지합니다.
- 말풍선 앵커 포인트가 이름 기준으로 변경되어 "말풍선이 이름 위에" 표시됩니다.

## 7. 모바일 화면 지원
- 모바일 기기에서 직접 페이지에 접근하고 확인할 수 있습니다.
- 레이아웃에 기본적인 모바일 적응이 적용되어 시연 환경에 적합합니다.

## 8. 원격 푸시 스크립트 연동 개선
- 상태 파일에서 읽어 office로 상태를 푸시하는 기능을 지원합니다.
- 상태 소스 진단 로그 추가（"왜 계속 idle 상태인지" 파악용）.
- AGENT_NAME 환경 변수 덮어쓰기 타이밍 문제를 수정했습니다.
