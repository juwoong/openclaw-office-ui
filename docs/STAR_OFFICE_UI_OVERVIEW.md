# Star Office UI — 기능 설명（Overview）

Star Office UI는 "픽셀 사무실" 시각화 인터페이스로, AI 어시스턴트/여러 OpenClaw 방문자의 상태를 웹 페이지（모바일 포함）에서 볼 수 있는 작은 사무실 씬으로 렌더링합니다.

## 화면에서 볼 수 있는 것
- 픽셀 사무실 배경（부감 시점）
- 캐릭터（Star + 방문자）가 상태에 따라 다른 구역으로 이동
- 이름과 말풍선（bubble）이 현재 상태/생각을 표시（커스텀 매핑 가능）
- 모바일에서도 표시 가능（포트폴리오 전시/라이브 스트리밍/외부 시연에 적합）

## 핵심 기능

### 1) 단일 Agent（로컬 Star）상태 렌더링
- 백엔드가 `state.json`을 읽어 `GET /status` 제공
- 프론트엔드가 `/status`를 폴링하여 `state`에 따라 Star의 구역 렌더링
- `set_state.py`를 제공하여 상태를 빠르게 전환

### 2) 다중 방문자（멀티 OpenClaw）사무실 입장
- 방문자가 `POST /join-agent`로 입장하여 `agentId`를 받음
- 방문자가 `POST /agent-push`로 자신의 상태를 지속 전송
- 프론트엔드가 `GET /agents`로 방문자 목록을 가져와 렌더링

### 3) Join Key（접속 키）메커니즘
- 고정 재사용 가능 join key 지원（예: `ocj_starteam01~08`）
- 각 key의 동시 접속 상한 지원（기본 3）
- "누가 사무실에 들어올 수 있는지"와 "한 key로 동시에 몇 명이나 입장 가능한지" 제어에 편리

### 4) 상태 → 구역 매핑（통일 로직）
- idle → breakroom（휴식 구역）
- writing / researching / executing / syncing → writing（작업 구역）
- error → error（장애 구역）

### 5) 방문자 애니메이션 및 성능 최적화
- 방문자 캐릭터가 애니메이션 스프라이트 사용
- WebP 리소스 지원（용량 더 작고, 로딩 더 빠름）

### 6) 이름/말풍선 겹침 없는 레이아웃
- 실제 방문자와 demo 방문자 분리 로직
- demo가 아닌 방문자의 이름과 말풍선이 전체적으로 위로 이동
- bubble이 이름 위에 고정되어 이름 가림 방지

### 7) Demo 모드（선택 사항）
- `?demo=1`일 때만 demo 방문자 표시（기본적으로 표시 안 함）
- demo와 실제 방문자가 서로 간섭하지 않음

## 주요 인터페이스（Backend）
- `GET /`：프론트엔드 페이지
- `GET /status`：단일 agent 상태（구버전 호환）
- `GET /agents`：다중 agent 목록（방문자 렌더링용）
- `POST /join-agent`：방문자 입장
- `POST /agent-push`：방문자 상태 전송
- `POST /leave-agent`：방문자 퇴장
- `GET /health`：헬스 체크

## 보안 및 개인정보 주의 사항
- 개인정보를 `detail`에 입력하지 마세요（렌더링되어 가져올 수 있으므로）
- 오픈소스 공개 전 반드시 정리: 로그, 실행 상태 파일, join keys, 터널 출력 등

## 아트 에셋 사용 선언（필수）
- 코드는 오픈소스 가능하나, 아트 소재（배경, 캐릭터, 애니메이션 등）의 저작권은 원작자/스튜디오에 있습니다.
- 아트 에셋은 학습과 시연 용도만 허용, **상업적 이용 금지**.
