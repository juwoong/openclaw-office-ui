# Star Office UI — 프로젝트 단계 요약（2026-03-01）

## 1. 오늘 작업 요약

오늘 주로 두 가지 주요 작업을 완료했습니다:

1. **다중 OpenClaw의 사무실 입장 기능 안정화**
2. **모바일 화면 시연 기능 완성**

그리고 "방문자 상태 동기화 불안정" 문제를 중심으로 여러 차례 점검하여 연결 문제와 현재 완전히 해결되지 않은 사항들을 명확히 했습니다.

---

## 2. 완료된 기능（외부 설명 가능）

### 1) 다중 Agent 입장 및 표시
- 여러 원격 OpenClaw가 `join-agent`를 통해 사무실에 입장할 수 있습니다.
- 각 방문자는 독립적인 `agentId`, 이름, 상태, 구역, 애니메이션을 가집니다.
- 씬은 `/agents`를 기반으로 방문자를 동적으로 생성, 업데이트, 제거합니다.

### 2) 고정 재사용 가능 Join Key 메커니즘
- 일회성 key를 고정 재사용 가능 key로 변경: `ocj_starteam01` ~ `ocj_starteam08`.
- "사용 즉시 재사용 불가" 차단 로직을 제거하여 장기 재사용을 지원합니다.
- 동시 접속 상한 설정（`maxConcurrent`）추가, 기본적으로 각 key당 3개 동시 접속 제한.

### 3) 동시 접속 제한 수정（핵심）
- 4개 동시 접속이 여전히 통과되는 근본 원인이 백엔드 경쟁 조건（race condition）임을 발견.
- `join-agent` 임계 구역에 잠금 + 잠금 내 상태 재읽기를 추가하여 수정 후 부하 테스트 통과:
  - 처음 3개 200
  - 4번째 429

### 4) 방문자 애니메이션 및 성능 최적화
- 방문자 애니메이션이 정적 별에서 픽셀 애니메이션 스프라이트로 변경.
- `guest_anim_1~6`이 `.webp`로 변환되어 로딩 용량이 현저히 감소.
- 프론트엔드 사전 로딩 및 렌더링 리소스가 webp 우선으로 전환.

### 5) 상태 → 구역 매핑 통일
- 규칙 통일:
  - `idle -> breakroom`
  - `writing/researching/executing/syncing -> writing`
  - `error -> error`
- 방문자 bubble 텍스트가 상태에 따라 매핑되어 더 이상 구역과 분리되지 않음.

### 6) 이름과 말풍선 레이어/위치 최적화
- demo가 아닌 방문자의 이름, 말풍선 위치가 위로 이동하여 겹침 감소.
- 방문자 말풍선 앵커가 이름 기준 상대 계산으로 변경되어 "말풍선이 이름 위에" 표시 보장.
- demo와 실제 방문자 경로가 구분되어 서로 간섭하지 않음.

### 7) 모바일 화면 시연
- 현재 UI가 모바일에서 접근 및 시연 가능하여 발표와 외부 공유에 적합.
- 핵심 컨트롤 레이아웃이 정리되어 모바일에서 기본적으로 사용 가능.

## 3. 현재 완전히 해결되지 않은 사항（솔직한 공개）

### 방문자 "실제 상태 안정적 동기화" 문제가 여전히 간헐적으로 불일치 발생
연결이 여러 차례 검증되었지만（writing으로 작업 구역 입장, idle로 휴식 구역 복귀 가능）, 실제 운영에서 다음과 같은 사례가 발생했습니다:
- 로컬 스크립트가 계속 idle 상태를 전송（구버전 스크립트 / 잘못된 상태 소스 읽기）
- 403 미승인（오프라인 상태 복구 / 이전 agentId 캐시 문제）
- 화면 앞에서 나가면 leave-agent 트리거 후 캐릭터 소멸

> 결론:
> - "메커니즘 가능, 연결 통과" 검증 완료;
> - "엔드투엔드 지속적 안정화"는 계속 보완이 필요（특히 방문자측 실행 스크립트 버전 통일, 상태 소스 통일, 상주 전략 통일）.

## 4. 오늘 새로 추가/수정된 파일（핵심）

- `backend/app.py`
  - join 동시 접속 제한 잠금 수정
  - offline/approved 인증 흐름 로직 조정（복구 용이화）
- `join-keys.json`
  - 고정 key + `maxConcurrent: 3`
- `frontend/index.html`（및 관련 렌더링 로직）
  - 방문자 애니메이션, 이름과 말풍선 위치 최적화
  - 상태 텍스트 매핑 조정
- `office-agent-push.py`（다중 버전 병렬 디버깅）
  - 상태 소스 진단 로그 추가
  - 환경 변수 덮어쓰기 로직 추가
  - AGENT_NAME 읽기 타이밍 문제 수정

## 5. 외부 오픈소스 공개 전 권장 설명（권장 문안）

> Star Office UI는 시각화 다중 Agent 픽셀 사무실입니다:
> 여러 OpenClaw 원격 접속, 상태 기반 위치 렌더링, 방문자 애니메이션, 모바일 접근을 지원합니다.
> 프로젝트는 현재 다중 Agent 주요 연결 및 UI 기능이 완성되었으며; 상태 동기화 안정성은 지속적으로 최적화 중입니다.
